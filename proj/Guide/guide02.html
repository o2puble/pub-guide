HTML 규칙

HTML 필수 규칙

  • 아래의 필수 규칙은 준수하지 않을 시 문서가 정상적으로 보이지 않을 수 있습니다.
  • 웹 표준 가이드를 준수하여 퍼블리싱하는 것을 기본으로 한다.
  • 모든 HTML 페이지 시작 지점에 공백없이 HTML5 문서타입을 선언한다.
  • HTML 문서에 맞게 lang 값을 작성한다.
  • HTML 태그 사용시 여는태그와 닫는태그는 필수로 작성한다.(닫는태그가 없는 태그는 제외)
  • 하나의 ID는 페이지에서 한번만 사용 한다.
  • inline 태그 안에는 block 태그가 들어갈 수 없다.
  • 웹표준 검사는 http://validator.w3.org에서 확인하거나 또는 Chrome 확장 프로그램을 설치하여 확인 한다.

마크업 유의사항

  • HTML 속성(attr) 값은 큰 따옴표("")를 사용한다.
  • 들여쓰기는 공백문자 4개를 사용한다.
  • html 문법상 <p> 태그 안에는 p, div , ul등의 block요소를 넣을 수 없다. (display:inline 속성을 적용해도 사용불가)
  • <p> 태그를 페이지 레이아웃을 잡는데 사용하지 않는다.
  • <p> 태그는 최하위 레벨의 내용에만 사용한다.
  • 구조 작성 시 한눈에 파악 가능하도록 작업한다.
  • img, input, label 등등의 태그들은 부모 태그(div,span 등등)를 한번 더 감싸서 수정사항이 있을 때 부모 요소로 컨트롤 할 수 있게 한다.
  • script는 <head></head> 내에 선언하는 것을 원칙으로하며 단, 성능상의 이슈가 있으면 개발 부서와 협의하여 선언 위치를 변경 할 수 있다.
  • html 태그에 inline style은 사용하지 않는 것을 원칙으로 하지만 js로 스타일을 적용하거나 개발 환경에 따라 스타일이 변하는 경우에 한해서 html에 inline style로 작성할 수 있다.

기능에 맞는 태그 사용

<a> tag

  • 내부 페이지 이동, 외부 사이트 이동은 <button> 태그가 아닌 <a> 사용한다.
  • 빈 링크의 href 값은 #none;으로 통일한다.
  • 클릭시 링크 방지 처리 예시
    But 페이지 이동이 필요 없는 경우 <a> 태그 보다 <button> 태그 사용을 우선한다.
  • GNB 로고에는 Home 화면으로 가는 링크를 제공한다.

<button> tag

  • 페이지 내에서 동작하는 기능은 <button> 태그 사용하며, <a> 사용은 지양한다.
  • button 태그에는 type을 반드시 선언 한다. (브라우저별로 기본값이 다름)
  • 모양은 button 이지만 기능이 링크 또는 label 인 경우에는 <button> 사용하지 않는다.
  • 사용 예시 : form 관련 버튼, toggle, tab, etc...

<input> tag

  • form값 전송을 위한 기능은 submit 타입의 input 또는 button 태그 사용한다.
  • disabled, readonly chekced, value, placeholder 등의 속성들을 정확히 이해하고 사용한다.

<h1> ~ <h6> tag

  • 헤딩요소에 스타일을 지정할때는 특정 요소가 아닌 클래스로 지정한다.
    (SEO개선을 위해 헤딩요소가 변경되는 경우가 있음) 
                                    .box_header h2 {color:red;} (X)

                                .box_header .heading {color:red;} (O)
  • 헤딩요소 사용시 h1 ~ h6 순서대로 작성한다.

<iframe> 요소

  • iframe 사용은 페이지 성능에 영향을 주기 때문에 가급적 사용을 하지 않는다.
  • title 속성은 반드시 제공되어야 하며 빈 값을 사용하지 않는다.
  • iframe을 포함하는 페이지의 도메인과 iframe에서 불러오는 페이지의 도메인이 다르면, 크로스 도메인 설정을 위해 별도의 소스 코드가 추가되어 성능의 영향을 줄 수 있다.
  • iframe의 높이값이 iframe에서 불러오는 콘텐츠의 따라 유동적이어야 한다면 별도의 소스코드가 추가되어 성능에 영향을 줄 수 있다.
  • iframe은 가장 마지막으로 로딩되기 때문에 페이지 로딩 초기에는 화면이 비어 보일 수 있다.

<img> 요소

  • alt 속성은 반드시 제공되어야 하며 의미없는 이미지에 한하여 빈 값을 허용한다.
  • 대체 텍스트 사용으로 일부 브라우저에서 말 풍선이 나타나는 것을 막을 필요가 있는 경우 title 속성의 빈 값을 제공한다.
  • 벡터 이미지(svg 등)가 아닌 경우 2배수 이미지 사용을 원칙으로 한다.
