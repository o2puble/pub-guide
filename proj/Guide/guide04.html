아이디/클래스 네이밍 규칙

아이디/클래스 네이밍 규칙
규칙 예시
클래스 작성시 단어를 구분 할때 하이픈(-)만을 사용해서 구분하며 언더바(_)나 카멜케이스는 사용 하지 않는다. 잘못된 예 .top_nav_box
.guideComponentHeader
올바른 예 .top-nav-box
.guide-component-header
클래스 명을 최대한 자세히 적고 너무 길어지는 경우는 자식 요소에서 부모 클래스 단어의 첫글자만 사용하여 줄인다. 부모 .guide-component-header
.guide-component-body
자식 .gch-tit-box
.gcb-info-list
아이디/클래스 이름은 용도를 나타내는 단어를 사용하되, 위치/크기/색상 등의 스타일을 특정하는 단어를 지양한다. 잘못된 예 #right-section
#thumb-300
올바른 예 #aside-section
#thumb-large
이름은 영문 대문자, 숫자, 특수문자로 시작할 수 없다. 예외 적으로 _(언더바)는 클래스명으로 시작하는 경우도 있음 잘못된 예 #Table
.@email
올바른 예 #table
.email
이름은 영문 대문자, 숫자, 특수문자로 시작할 수 없다. 예외 적으로 _(언더바)는 클래스명으로 시작하는 경우도 있음 예시 .list-benefit
.list-benefit02
.list-benefit03
네이밍의 조합은 '형태-의미+순서-상태'을 기본 순서로 사용한다. 예시 .btn-confirm02-on
.list-benefit02-active
형태를 나타내는 접두사를 이용해 클래스명을 사용한다. 예시 box-
title-
list-
table-
btn-
tab-
상태별 접미사를 이용해 직관적으로 확인할 수 있는 클래스명을 사용한다. 예시 on
off
active
종속 → 형태-요소-v + n → 의미전달이 확실하지 않으므로 해당 클래스에 대한 스타일 작성 시 어떤 부분이 바뀌는지에 대한 주석이 필요 예시 box-list-v1
box-list-v2
독립 → 형태-요소0n → type 개념으로 나뉜 클래스 예시 box-list01
box-list02
독립 > 종속 → 형태-요소0n-v+n 예시 box-list02-v1
box-list02-v2
